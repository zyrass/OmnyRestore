## 1. Executive Summary

OmnyRestore is a **professional web platform** combining:
- A **public showcase** (landing page, portfolio, CTA)
- A **client space** (order submission, real-time tracking, payment, download)
- An **admin back office** (order management, AI-enhanced photo upload, delivery)

The **key differentiator** is the AI-powered restoration pipeline:
the admin uses a precision-crafted ChatGPT prompt to upscale and restore photos to 8K quality.
The client sees a watermarked preview, pays, and downloads the full-resolution ZIP.

This eliminates the classic "pay before work" vs "work before pay" dilemma:
since AI restoration takes minutes, the platform can show results before payment
while maintaining 100% revenue security.

## 4. Business Flows — Sequence Diagrams

### 4.1 Client Order Submission

```mermaid
sequenceDiagram
    actor Client
    participant Browser
    participant Livewire as Livewire OrderCreateForm
    participant OpenAI as OpenAI GPT-4o Vision
    participant Laravel
    participant DB as PostgreSQL

    Client->>Browser: Sélectionne 1-10 photos
    Browser->>Livewire: wire:model.live (updatedPhotos)
    Livewire->>OpenAI: analyze(photo) × N
    OpenAI-->>Livewire: level (light|medium|heavy) + confidence
    Livewire-->>Browser: Verdict IA + prix (1|2|5 € TTC/photo)

    opt Coupon disponible
        Client->>Browser: Saisit code réduction
        Livewire->>DB: CouponService::apply(code, baseHtCents)
        DB-->>Livewire: {valid, discount_cents, message}
        Livewire-->>Browser: Récapitulatif mis à jour (HT/TVA/TTC)
    end

    Client->>Browser: Soumet la commande
    Browser->>Livewire: submit()
    Livewire->>DB: INSERT orders (base_price_cents, total_price_cents, coupon_code, discount_cents)
    Livewire->>DB: CouponService::confirm() → INCREMENT used_count
    Livewire->>DB: INSERT media (collection = originals)
    Livewire-->>Browser: Redirect /client/orders/{id} (status: PENDING)
```

### 4.2 Admin Processing (AI-Assisted Restoration)

```mermaid
sequenceDiagram
    actor Admin
    participant Browser
    participant Livewire as Livewire AdminOrderShow
    participant DB
    participant Queue

    Admin->>Browser: Ouvre commande depuis dashboard
    Browser->>Livewire: mount(order)
    Admin->>Browser: "Prendre en charge"
    Livewire->>DB: UPDATE status = IN_PROGRESS

    Note over Admin,Livewire: Admin restaure les photos via IA externe (ChatGPT)

    Admin->>Browser: Upload photos restaurées
    Livewire->>DB: INSERT media (collection = retouched)

    opt Contrôle qualité photo
        Admin->>Browser: "Rejeter" photo de mauvaise qualité
        Livewire->>DB: media.custom_properties.is_rejected = true
        Livewire->>DB: recalcPriceFromActivePhotos()
        Note right of DB: total_price_cents recalculé
        Note right of DB: coupon réappliqué si présent
    end

    Admin->>Browser: "Marquer comme terminé"
    Livewire->>DB: UPDATE status = DONE
    Livewire->>Queue: Dispatch GenerateOrderZipJob
    Queue->>DB: getMedia('retouched').filter(not rejected)
    Queue->>DB: INSERT order_deliveries (zip_path)
```

### 4.3 Payment & Download (Stripe Checkout)

```mermaid
sequenceDiagram
    actor Client
    participant Browser
    participant Laravel
    participant Stripe
    participant DB
    participant S3
    participant Queue
    participant Mail

    Client->>Browser: Click "Pay & download"
    Browser->>Laravel: POST /client/orders/{id}/checkout
    Laravel->>Stripe: Create Checkout Session (line items, metadata)
    Stripe-->>Laravel: session_url
    Laravel-->>Browser: Redirect to Stripe Checkout

    Client->>Stripe: Enter card details + 3DS2 SCA
    Stripe->>Laravel: POST /webhook/stripe (payment_intent.succeeded)

    Note over Laravel: Verify HMAC-SHA256 webhook signature

    Laravel->>DB: UPDATE payment_status = paid, paid_at = now()
    Laravel->>Queue: Dispatch GenerateSignedDownloadUrlJob
    Queue->>S3: Generate pre-signed URL (TTL: 48h)
    Queue->>DB: UPDATE order_deliveries.signed_url
    Queue->>Mail: Email client — Download link (48h validity)

    Client->>Browser: Click link in email or client space
    Browser->>Laravel: GET /client/orders/{id}/download
    Laravel->>DB: Verify payment_status = paid (FilePolicy)
    Laravel->>S3: Redirect to signed URL
    S3-->>Browser: ZIP file download
    Laravel->>DB: INSERT audit_logs (action = DOWNLOAD)
```

## 5. Order State Machine

```mermaid
stateDiagram-v2
    [*] --> PENDING : Client submits order + photos

    PENDING --> IN_PROGRESS : Admin takes charge
    PENDING --> CANCELLED : Admin or client cancels

    IN_PROGRESS --> DONE : Admin uploads AI-restored photos
    IN_PROGRESS --> CANCELLED : Admin cancels (unrestorable photos)

    DONE --> PAID : Stripe webhook payment_intent.succeeded
    PAID --> DOWNLOADED : Client downloads ZIP

    CANCELLED --> [*]
    DOWNLOADED --> [*]

    note right of PENDING
        Triggers:
        → Email to admin (new order)
        → Dashboard badge increment
    end note

    note right of IN_PROGRESS
        Triggers:
        → Email to client (order started)
        → Admin accesses original photos
    end note

    note right of DONE
        Triggers:
        → Async ZIP generation job
        → Watermarked preview generated
        → Payment email sent to client
    end note

    note right of PAID
        Triggers:
        → Signed download URL generated (48h TTL)
        → Download email sent to client
        → Audit log: PAYMENT
    end note

    note right of DOWNLOADED
        Triggers:
        → Audit log: DOWNLOAD
        → (Future) Media cleanup scheduled
    end note
```

## 7. Payment Flow (Stripe)

```
Payment method: Stripe Checkout (hosted by Stripe)
Advantages   : PCI-DSS offloaded, native 3DS2/SCA, European compliance built-in
```

```mermaid
graph LR
    A[Client clicks Pay] --> B[Laravel creates<br/>Checkout Session]
    B --> C[Stripe Checkout<br/>Hosted Page]
    C --> D{3DS2<br/>Validation}
    D -->|Success| E[Stripe fires webhook<br/>payment_intent.succeeded]
    D -->|Failure| F[Client notified<br/>Payment failed]
    E --> G[Laravel webhook handler<br/>HMAC-SHA256 verified]
    G --> H[DB: payment_status = paid]
    H --> I[Async: Generate signed URL]
    I --> J[Email: Download link 48h]
```

**Stripe `.env` configuration:**
```bash
STRIPE_KEY=pk_test_xxx
STRIPE_SECRET=sk_test_xxx
STRIPE_WEBHOOK_SECRET=whsec_xxx
```

**Webhook events to listen:**
- `payment_intent.succeeded` → unlock download
- `payment_intent.payment_failed` → notify client
- `charge.refunded` → update `payment_status = refunded`

## 8. ZIP Delivery Architecture

```mermaid
graph TB
    A[Admin marks order DONE] --> B[GenerateOrderZipJob dispatched<br/>via Redis queue]
    B --> C[Fetch all retouched media<br/>from Spatie Media Library]
    C --> D[PHP ZipArchive:<br/>add each file to ZIP in memory]
    D --> E[Upload ZIP to S3<br/>disk:deliveries/ORD-2026-XXXX.zip]
    E --> F[INSERT order_deliveries<br/>zip_disk + zip_path]
    F --> G[GenerateSignedDownloadUrlJob]
    G --> H[S3 presigned URL<br/>TTL: 48 hours]
    H --> I[UPDATE order_deliveries.signed_url]
    I --> J[Email client: Download link]
```

**File access policy (IDOR + payment prevention):**
```
GET /client/orders/{id}/download
    → Authenticate (auth middleware)
    → OrderPolicy::download() — verifies user owns the order
    → Checks: payment_status = paid
    → If signed_url expired: regenerate it on the fly
    → 302 Redirect to fresh presigned S3 URL
    → INSERT audit_logs (action = DOWNLOAD)
```

## 9. AI Restoration Pipeline

The AI restoration is performed using a precision-crafted prompt
submitted to **ChatGPT GPT-4o** (or OpenAI API for future automation):

```
Act as a high-precision visual data restoration algorithm.
Mission: correct compression artifacts and motion blur to recover original clarity.

TECHNICAL INSTRUCTIONS:
1. Perform intelligent upscale to 8K resolution without altering base vectors.
2. Reconstruct surface textures (micro-details, pores, fibers) from existing luminance data.
3. Stabilize edge sharpness and optimize global contrast for "Studio Professional" rendering.
4. Strictly preserve geometry, facial features, and source lighting.

Expected result: An ultra-sharp, photorealistic, clean version of the provided image,
with zero structural modifications.
```

**Phase 1 (MVP):** Admin-assisted (manual prompt + ChatGPT UI)

**Phase 2 (v1.2.0 roadmap):** Automated via OpenAI API
```mermaid
graph LR
    Upload[Admin uploads<br/>original photo] --> Job[AIRestorationJob<br/>dispatched to queue]
    Job --> API[POST openai.com/v1/images/edits<br/>model: gpt-4o<br/>prompt: restoration template]
    API --> Result[Restored image URL]
    Result --> S3[Upload to S3<br/>collection: retouched]
    S3 --> Preview[Generate watermarked preview]
    Preview --> Notify[Notify client]
```

## 10. Security Architecture (OWASP Top 10)

| # | Vector | Countermeasure |
|---|---|---|
| A01 | Broken Access Control | `OrderPolicy`, `FilePolicy` — ownership check on every request |
| A02 | Cryptographic Failures | HTTPS TLS 1.3, S3 SSE-AES256, bcrypt passwords |
| A03 | Injection | Eloquent ORM + Query Builder only — zero raw SQL |
| A04 | Insecure Design | Payment-before-download, webhook HMAC verification |
| A05 | Security Misconfiguration | `.env` in `.gitignore`, S3 private buckets, no directory listing |
| A06 | Vulnerable Components | `composer audit` + `npm audit` in CI, Dependabot |
| A07 | Auth Failures | `throttle:60,1` on auth, email verification required, 2FA admin |
| A08 | Software Integrity | `composer.lock` + `package-lock.json` version-pinned |
| A09 | Logging & Monitoring | `AUDIT_LOGS` table + Laravel Telescope + Slack alerts |
| A10 | SSRF | No user-controlled HTTP requests; S3 signed URLs are server-generated |
